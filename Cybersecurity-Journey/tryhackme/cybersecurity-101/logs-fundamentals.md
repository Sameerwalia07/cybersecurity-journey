## 🧠 In My Own Words

**Logs** are the digital footprints left behind by any activity on a system
— whether that activity is completely normal or carries malicious intent.
Because of this, logs make it much easier to **trace down what happened**
and **who (or what) was behind it**.

**Use cases for logs** go beyond just spotting attacks:
- **Security event monitoring** — watching for suspicious activity in
  real time
- **Incident investigation and forensics** — reconstructing what happened
  after an incident
- **Troubleshooting** — diagnosing why something isn't working correctly
- **Performance monitoring** — tracking system health and behaviour over
  time
- **Auditing and compliance** — proving that required processes/controls
  were actually followed, often for legal/regulatory reasons

**Types of logs:**
- **System logs** — record OS-level events and activity
- **Audit logs** — record actions taken, often for compliance/accountability
- **Security logs** — record security-relevant events (logins, permission
  changes, etc.)
- **Network logs** — record traffic and network-level activity
- **Access logs** — record who accessed what, and when
- **Application logs** — record events generated by specific applications

**Windows OS logs** specifically fall into three main categories:
- **System logs** — events from Windows and its core components
- **Security logs** — security-related events like logons and privilege use
- **Application logs** — events generated by installed applications

Windows log entries typically include:
- **Description** — details about what happened
- **Log Name** — which log category the entry belongs to
- **Logged** — the timestamp of when the event occurred
- **Event ID** — a specific numeric code identifying the type of event,
  which is extremely useful for quickly recognising and filtering for known
  event types

The room also covered **web server log analysis using Linux commands**
directly from the terminal:
- `cat` — print the full log file
- `grep` — search for specific patterns/keywords within the log
- `less` — view a log file page by page, without loading the whole thing at
  once — useful for very large log files
  - **Spacebar** — move to the next page
  - **b** — move back to the previous page
